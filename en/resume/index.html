Shipped Projects

THORN — Parametric Lighting System
View Project

Thorn is a Red Dot Award–winning modular ceiling light by QZENS, built from three primary component types. Taking it to production meant solving a combinatorial problem: how do you document every valid configuration without manually tracking hundreds of part combinations? I built a parametric generation system in Blender using Geometry Nodes and the Blender API that encodes the assembly rules directly into the node graph. Length, density, and branching frequency can all vary; the system handles the constraints. A custom export add-on reads the geometry at output time and compiles the component counts, part types, and configuration data into fabrication-ready documentation automatically. The result is a design-to-production pipeline where the 3D model and the manufacturing docs stay in sync. Change the design, regenerate the export and that's it.

0 Days Without Accidents
View on Devpost

An award-winning endless runner, 1st Place at the Devpost Hackathon. You play a new facility technician at Station 496 who accidentally triggers an explosion and has to escape through an unstable building while collecting scattered succulents. The core technical challenge was level generation. I wanted each run to feel structurally different without producing dead ends or broken geometry. Rather than implementing wave function collapse directly, I adapted its core ideas — adjacency rules, compatibility states, weighted randomness — into a modular tile-based generation system. Each tile segment carries its own spatial and collision constraints, so the runtime assembles levels from those rules rather than from a fixed layout. The hackathon scope forced real decisions about what to cut. The generation logic ended up small but deterministic, it was compact enough to tune quickly, robust enough not to break under time pressure.

NLE Batch Editor
View on Github

NLA Batch Editor is a GNU-licensed, open-source Blender add-on for working faster inside the NLA Editor. I built it originally for my own animation workflow, then cleaned it up and released it when I realized the missing functionality wasn't just my problem. It adds batch pushdown, track and strip management, advanced selection tools, and bulk cleanup operations: the kind of repetitive manual steps that pile up fast on large rigs or multi-character scenes. The add-on is open for contributions on GitHub.

Azuki: Elemental XR Experience
View Project

An augmented reality experience built to promote Azuki's merchandise through effects tied to the brand's character universe. I designed four elemental VFX systems. Each one tracks the user's body position in real time, orbiting and reacting as they move. Each effect runs on custom GPU shaders that drive both the particle systems and vertex displacement. The swirl speed, particle density, and reaction curves are all procedural, so the effects feel organic rather than looping. The main constraint was mobile XR performance: keeping the visuals dense enough to read well while staying within draw call and particle count budgets required ongoing tradeoffs throughout development.
